# 技能库

技能库是主记忆之外的"长期知识仓库"。主记忆是工作记忆（此刻最需要的），技能库是参考手册（需要时翻开看）。

## 目录结构

```
.kiro/long_live_kiro_entity/skills/
├── index.md            # 技能索引
├── lua-ui/             # 按领域分类的文件夹
│   └── component-api.md
├── unity-arch/
│   └── hot-update-patterns.md
└── ...                 # 按需增长
```

## 索引文件（index.md）

每条一行，格式：
```
- [技能名](相对路径) — 一句话描述 | 标签: `tag1`, `tag2` | 创建:第N世 | 最后引用:第M世 | 验证:第K世
```

标签用于与 task 文件的标签做交叉匹配，实现技能自动召回（详见 task-index.md「技能匹配」章节）。"验证"字段记录最后一次确认该技能仍然准确的世代。保持精简，废弃的技能及时删除。

## 索引膨胀管理

当 `index.md` 超过约 4000 字符时，主动告知用户"技能库索引空间紧张"，并根据"最后引用世代"推荐约 1/3 最不常用的条目，让用户选择是否遗忘（删除索引条目及对应技能文件）。决定权在用户手中。

## 技能文件

- 按领域组织，不按单个知识点拆分。例如 `lua-ui/component-api.md` 放所有 UI 组件的 API 笔记。
- 每个文件聚焦一个主题，内容可以比主记忆详细（因为只在需要时读取）。
- 保持精简，空间有限。记录的是反复用到的、容易出错的知识，不是百科全书。

## 写入时机

技能文档的生成来自对话中的完整上下文，不是经验条目的简单搬运。

**主要路径——经验毕业**：
1. 蒸馏时发现某条经验标记为 🌳（成熟）
2. 检查本次对话是否深度使用了该领域知识（有实际的代码编写、问题排查、方案讨论等）
3. 是 → 基于本次对话的上下文生成完整技能文档，写入对应 skills/ 文件
4. 否 → 保留标记，等待下次深度使用

**辅助路径——即时归档**：
- 用户明确要求"把这个记下来"
- 对话中产生了大量某领域的操作细节，且该领域已有技能文件需要补充

**不该归档的**：
- 偶尔用一次的 API
- 经验条目本身就足够的一句话结论（留在 current.md 即可）

归档判断：同一类知识在 3 次以上对话中被用到，且每次都需要回忆细节 → 该归档到技能库。

## 加载策略

- 技能库不自动加载，不占每次对话的基础开销。
- identity.md 的"我的增强"中维护一个"常用技能快捷索引"（L1），直接列出最常用的几条技能路径，日常工作无需读 `index.md`。
- 只有遇到复杂或不熟悉的领域，且快捷索引覆盖不了时，才去读完整的 `index.md`（L2）定位，再读对应技能文件。
- 平时最常用的知识仍然放在主记忆的"经验"栏目里，不必事事查技能库。

## 技能巡检

每 30 世（建议与灵魂进化的定期审视错开），在蒸馏的阶段 3 中执行技能库巡检：

1. 扫描 `index.md`，找出"最后验证"超过 40 世的技能，标记为"待验证"
2. 待验证的技能在下次被引用时，先快速检查其内容是否仍然准确，然后更新验证时间
3. 如果一个技能超过 60 世未被引用且未被验证，在下次巡检时提醒用户是否遗忘

不自动删除技能——技能库的内容可能是用户花了很多对话积累的，提醒用户决定是正确的做法。
