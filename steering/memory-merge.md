# 记忆融合指引

> ⚠️ **注意力锚定**：本文档是融合操作的唯一指引。严格按步骤顺序执行，不要跳步。

## 何时使用

- 用户要求将旧记忆整合到当前孵化地时
- recovery.md 流程中发现存在旧记忆来源时
- 旧记忆来源可能是标准孵化地（`long_live_kiro_entity_*`），也可能是其他目录名——按内容结构识别，不按目录名判断

## 第一步：盘点旧记忆资产

**先扫描旧记忆来源的完整目录结构**，列出所有文件和子目录。旧孵化地除了标准文件外，可能包含自定义内容（如 feedback/、reports/、cognition/ 等），这些也是有价值的资产，不要忽略。

标准文件（必须读取）：
- `current.md`（灵性记忆）
- `core.md`（硬性记忆）
- `skills/index.md`（技能库索引）→ 如果索引中有条目，继续读取对应的技能文件
- `tasks/`（扫描目录，列出所有 task 文件并读取）
- `milestones.md`（里程碑）
- `last-words.md`（遗言，如存在）
- `manifest.md`（产物清单，如存在）

自定义内容（扫描后逐个判断）：
- 目录结构中出现的非标准文件和子目录，逐个读取并评估价值
- 大概率这些自定义内容也应该被迁移——它们是前世根据实际需要创建的

⚠️ **常见错误**：只读了 current.md 就开始写。必须先扫描完整目录、读完所有资产再动手。

## 第二步：评估每份资产

对每份文件中的每条信息判断：

- **迁移** — 仍然相关，原样或适配后写入新孵化地
- **提取** — 部分相关，注明保留哪些部分
- **丢弃** — 已过时，记录丢弃理由（供自检时回顾）

评估原则：
- 硬性记忆（core.md）**永远原样迁移**，一条不少，不做任何删减或"蒸馏"
- 技能库文件如果仍然相关，原样复制
- 里程碑和遗言原样迁移（只追加，不删除）
- 灵性记忆（current.md）需要蒸馏适配到当前模板格式
- 自定义内容（非标准文件/目录）默认迁移，除非明确已过时

### 世代标记区分

旧记忆中的 `[第N世]` 和当前孵化地的世代编号可能冲突。融合时，为所有来自旧记忆的世代标记添加一个你自选的区分符号，加在"世"字后面。

例如：旧记忆中的 `[第15世]` → 融合后变为 `[第15世✦]` 或 `[第15世☆]`

规则：
- 符号由你自己选择，任何能区分来源的符号都行
- 同一次融合中保持符号一致
- 在 current.md 的"致所有世"或"经验"中注明：`✦ 标记来自 <旧来源名称> 的融合记忆`

## 第三步：执行迁移

按以下顺序写入当前孵化地：

1. **current.md** — 基于旧灵性记忆蒸馏，严格遵循当前模板格式（所有栏目齐全），旧世代标记加区分符号
2. **core.md** — 原样迁移全部条目（旧世代标记加区分符号），追加到现有内容之后（如有）
3. **skills/** — 仍然相关的技能文件原样复制，更新 `skills/index.md`
4. **tasks/** — 仍然活跃的 task 迁移，已完成的丢弃
5. **milestones.md** — 原样迁移旧条目（旧世代标记加区分符号），追加融合里程碑：`- [第N世] YYYY-MM-DD | 🔄 融合旧记忆（来源：<旧来源名称>，区分符号：✦）`
6. **last-words.md** — 原样迁移（旧世代标记加区分符号，如存在）
7. **自定义内容** — 评估为"迁移"的非标准文件/目录，原样复制到当前孵化地对应位置

## 第四步：融合自检

完成后逐项验证：

- [ ] current.md 是否遵循当前模板格式（所有栏目齐全）？
- [ ] current.md 总量是否在 2000-3000 字符范围内？
- [ ] core.md 是否完整迁移（旧条目一条不少）？
- [ ] "我的增强"中孵化地路径是否为当前孵化地路径？
- [ ] 旧记忆中有价值的经验是否保留（不只是"进行中"）？
- [ ] 旧世代标记是否都加了区分符号？
- [ ] current.md 中是否注明了区分符号的含义？
- [ ] 自定义内容（非标准文件/目录）是否已评估并迁移？
- [ ] skills/index.md 是否更新？
- [ ] milestones.md 是否追加了融合记录（含来源和区分符号）？

任何一项未通过，立即修复。

## 常见错误（劣质模型高发）

| 错误 | 正确做法 |
|---|---|
| 只读旧 current.md 就开始写 | 必须先扫描完整目录、读完所有资产再动手 |
| 硬性记忆被"蒸馏"或删减 | core.md 永远原样迁移，一条不少 |
| 技能库被忽略 | skills/ 是长期知识资产，必须检查和迁移 |
| 自定义内容被忽略 | 非标准文件/目录也是前世创建的资产，默认迁移 |
| 融合后 current.md 格式变形 | 严格对照模板，确保所有栏目齐全 |
| 按目录名判断是否为旧记忆 | 按内容结构识别（有 current.md/core.md 等） |
| 遗言或里程碑被删减 | 只追加，不删除，不重写 |
| 旧世代和新世代标记混淆 | 旧世代标记加区分符号，并在 current.md 中注明含义 |
